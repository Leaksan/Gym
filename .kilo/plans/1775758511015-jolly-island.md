# Plan: Pré-remplir les champs de la première page

## Objectif
Pré-remplir les champs suivants dans la première étape (Candidat) de l'application avec des valeurs par défaut, tout en les gardant modifiables par l'utilisateur :
- Champ "Session" : valeur par défaut "2026"
- Champ "Type d'examen" : valeur par défaut "Baccalauréat général"

## Étapes d'implémentation
1. **Examiner le code HTML** : Confirmer l'emplacement des champs dans `index.html`, spécifiquement dans la section `step1` (lignes ~619-621 pour Session et ~612-613 pour Type d'examen).
2. **Modifier les éléments input** :
   - Ajouter l'attribut `value="2026"` à l'input avec `id="s1_session"`.
   - Ajouter l'attribut `value="Baccalauréat général"` à l'input avec `id="s1_type_examen"`.
3. **Vérifier l'accessibilité** : S'assurer que les champs restent modifiables (les inputs text le sont par défaut).
4. **Tester les modifications** : Ouvrir l'application dans un navigateur pour confirmer que les valeurs par défaut apparaissent et peuvent être modifiées.
5. **Linting/Validation** : Si applicable, exécuter des outils de linting pour le HTML/JS.

## Considérations
- Les valeurs par défaut sont en français et correspondent aux placeholders existants.
- Aucune modification JavaScript n'est nécessaire car les inputs text conservent leur comportement par défaut.

## Risques potentiels
- Modifications HTML simples, faible risque de cassure.
